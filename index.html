Bulan 1 – 2
Rakit hardware hexapod
Prioritas utama. Pasang 18 servo WP23KG, frame, kabel, dan power supply LiPo.
Ukur spesifikasi fisik robot
Catat panjang tiap link, range sudut servo, distribusi berat untuk pembuatan URDF.
Pelajari konsep dasar RL
Pahami reward, state, action, policy, episode. Bisa dari YouTube (David Silver).
Coba Gymnasium + SB3
Latih PPO di environment sederhana (CartPole, HalfCheetah) sebagai warm-up.
Setup komunikasi Raspi–Arduino
Uji serial communication: Raspberry Pi 4 → Arduino ATmega → servo test.
Mulai Bab 1 & 2
Tulis pendahuluan dan tinjauan pustaka. Kumpulkan minimal 15 jurnal referensi.
NOTEHardware dan studi literatur berjalan paralel. Manfaatkan waktu rakit untuk baca jurnal dan tonton video pembelajaran RL.

Simulasi

Simulasi PyBullet
Bulan 3 – 4
Buat/adaptasi model URDF
Model URDF hexapod 18-DOF berdasarkan spesifikasi hardware nyata.
Buat custom Gymnasium env
Implementasi environment hexapod dengan PyBullet sebagai physics engine.
Rancang terrain tidak rata
Implementasi random heightmap dengan domain randomization untuk generalisasi.
Definisikan state space
Orientasi tubuh (roll/pitch/yaw), angular velocity, sudut 18 joint, arah target.
Definisikan action space
Continuous: delta sudut target untuk 18 servo secara bersamaan.
Rancang reward function v1
Forward reward + stability reward + survival reward + energy penalty.
Lanjutkan Bab 3
Tulis metodologi penelitian secara detail bersamaan dengan implementasi.
NOTEKualitas URDF sangat menentukan keberhasilan sim-to-real. Pastikan dimensi dan massa sesuai hardware nyata.

Algoritma PPO
Bulan 5 – 7
Training PPO pertama
Jalankan training awal, observasi apakah agent bisa berdiri dan bergerak maju.
Iterasi reward function
Tuning bobot reward berdasarkan perilaku agent. Ini proses paling iteratif.
Hyperparameter tuning
Eksperimen learning rate, clip range, n_steps, entropy coefficient.
Analisis learning curve
Monitor konvergensi reward, value loss, policy loss via TensorBoard.
Uji domain randomization
Evaluasi kemampuan generalisasi kebijakan terhadap variasi terrain berbeda.
Dokumentasi eksperimen
Catat setiap konfigurasi, hasil, dan observasi. Ini bahan utama Bab 4.
⚠ KEYIni fase terberat. Wajar jika reward function perlu direvisi 5–10 kali. Iterasi adalah prosesnya, bukan kegagalan.
